🧪 Write Unit Tests for Your App

Create unit tests to ensure your app functions correctly and catch bugs early 🔬

Write 3 unit tests under tests/test_*.py. The example structure here as inspiration PaperLM's test_basics.py

📝 Action Items:

  1. 📁 Set up testing structure: Create tests/ directory and install pytest (pip install pytest)
  2. 🔧 Write 3 core functionality tests: Here are some exmaples:
    • PDF files are correctly uploaded and processed
    • User registration works with valid credentials
    • Correct password leads to successful login
    • Logged-in users have the right documents saved
  3. 📤 Share your tests: Push your unit tests file to GitHub and share the GitHub URL link in this Discord thread

🤖 Test Your Large Language Models

Test your AI/LLM components to ensure they behave correctly and produce reliable outputs 🎯

Write 3 LLM tests under tests/test_*.py. Use this structure as inspiration PaperLM's test_prompts.py

📝 Action Items:

  1. 📁 Set up LLM testing: Create tests/ directory and create test_llm.py or test_prompts.py
  2. 🔧 Write 3 LLM behavior tests: Here are some examples:
    • Check if the LLM output matches expected content/format
    • Check if the output has the right structure (JSON, specific fields, etc.)
    • Check if the input prompt is processed correctly
    • Test prompt variations produce consistent results
  3. 📤 Share your tests: Push your LLM tests file to GitHub and share the GitHub URL link in this Discord thread

👍👎 Collect User Feedback on App Outputs

Add thumbs up/down feedback system and collect quality examples of your app's performance 📊

📝 Action Items:

  1. 👍👎 Add feedback system: Implement thumbs up/thumbs down buttons for user outputs in your app interface
  2. 📁 Create outputs folder: Set up an outputs/ directory to store feedback examples
  3. ✅ Collect 2 thumbs up outputs (save them in outputs/): Save 2 high-quality outputs that users rated positively (include input, output, and timestamp)
  4. ❌ Collect 2 thumbs down outputs (save them in outputs/): Save 2 problematic outputs that users rated negatively (include input, output, and reason for negative feedback)
  5. 📤 Share your feedback collection: Push your outputs folder to GitHub and share the GitHub URL link in this Discord thread
